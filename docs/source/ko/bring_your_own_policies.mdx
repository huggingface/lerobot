# 사용자 정의 정책 가져오기 (Bring Your Own Policies)

이 튜토리얼은 사용자 정의 정책(policy) 구현을 LeRobot 생태계에 통합하여, 자신만의 알고리즘을 사용하면서도 학습, 평가, 배포를 위한 LeRobot의 모든 도구를 활용할 수 있는 방법을 설명합니다.

## 1단계: 정책 패키지 생성

사용자 정의 정책은 LeRobot의 플러그인 관례를 따르는 설치 가능한 Python 패키지로 구성되어야 합니다.

### 패키지 구조

`lerobot_policy_` 접두사(중요!) 뒤에 정책 이름을 붙여 패키지를 생성합니다:

```bash
lerobot_policy_my_custom_policy/
├── pyproject.toml
└── src/
    └── lerobot_policy_my_custom_policy/
        ├── __init__.py
        ├── configuration_my_custom_policy.py
        ├── modeling_my_custom_policy.py
        └── processor_my_custom_policy.py
```

### 패키지 설정

`pyproject.toml`을 다음과 같이 설정합니다:

```toml
[project]
name = "lerobot_policy_my_custom_policy"
version = "0.1.0"
dependencies = [
    # 정책별 의존성을 여기에 작성하세요
]
requires-python = ">= 3.12"

[build-system]
build-backend = # 사용하려는 빌드 백엔드
requires = # 사용하려는 빌드 시스템
```

## 2단계: 정책 설정(Configuration) 정의

[`PreTrainedConfig`](https://github.com/huggingface/lerobot/blob/main/src/lerobot/configs/policies.py)를 상속하고 정책 타입을 등록하는 설정(Configuration) 클래스를 생성합니다:
다음은 시작을 위한 템플릿입니다. 정책의 아키텍처와 학습 요구사항에 맞게 매개변수와 메서드를 적절히 수정하세요.

```python
# configuration_my_custom_policy.py
from dataclasses import dataclass, field
from lerobot.configs import PreTrainedConfig
from lerobot.optim import AdamWConfig
from lerobot.optim import CosineDecayWithWarmupSchedulerConfig

@PreTrainedConfig.register_subclass("my_custom_policy")
@dataclass
class MyCustomPolicyConfig(PreTrainedConfig):
    """MyCustomPolicy를 위한 설정 클래스.

    Args:
        n_obs_steps: 입력으로 사용할 관측 스텝(observation step)의 수
        horizon: 액션 예측 구간
        n_action_steps: 실행할 액션 스텝(action step)의 수
        hidden_dim: 정책 네트워크의 은닉 차원
        # 여기에 정책별 매개변수를 추가하세요
    """

    horizon: int = 50
    n_action_steps: int = 50
    hidden_dim: int = 256

    optimizer_lr: float = 1e-4
    optimizer_weight_decay: float = 1e-4

    def __post_init__(self):
        super().__post_init__()
        if self.n_action_steps > self.horizon:
            raise ValueError("n_action_steps cannot exceed horizon")

    def validate_features(self) -> None:
        """입력/출력 feature 호환성을 검증합니다."""
        if not self.image_features:
            raise ValueError("MyCustomPolicy requires at least one image feature.")
        if self.action_feature is None:
            raise ValueError("MyCustomPolicy requires 'action' in output_features.")

    def get_optimizer_preset(self) -> AdamWConfig:
        return AdamWConfig(lr=self.optimizer_lr, weight_decay=self.optimizer_weight_decay)

    def get_scheduler_preset(self):
        return None

    @property
    def observation_delta_indices(self) -> list[int] | None:
        """데이터셋 로더가 observation마다 제공하는 상대 타임스텝 오프셋입니다.

        단일 프레임 정책의 경우 `None`을 반환합니다. 여러 과거 또는 미래 프레임을
        사용하는 시간적(temporal) 정책의 경우 오프셋 리스트를 반환합니다.
        예: stride 10으로 3개의 과거 프레임과 stride 10으로 1개의 미래 프레임인 경우 `[-20, -10, 0, 10]`.
        """
        return None

    @property
    def action_delta_indices(self) -> list[int]:
        """데이터셋 로더가 반환하는 액션 청크(action chunk)에 대한 상대 타임스텝 오프셋입니다.
        """
        return list(range(self.horizon))

    @property
    def reward_delta_indices(self) -> None:
        return None
```

        super().__init__(config, dataset_stats)

## 5단계: 패키지 초기화

패키지의 `__init__.py`에서 클래스들을 외부로 공개합니다:

```python
# __init__.py
"""LeRobot용 사용자 정의 정책 패키지."""

try:
    import lerobot  # noqa: F401
except ImportError:
    raise ImportError(
        "lerobot is not installed. Please install lerobot to use this policy package."
    )

from .configuration_my_custom_policy import MyCustomPolicyConfig
from .modeling_my_custom_policy import MyCustomPolicy
from .processor_my_custom_policy import make_my_custom_policy_pre_post_processors

__all__ = [
    "MyCustomPolicyConfig",
    "MyCustomPolicy",
    "make_my_custom_policy_pre_post_processors",
]
```

## 6단계: 설치 및 사용

### 정책 패키지 설치

```bash
cd lerobot_policy_my_custom_policy
pip install -e .
# 또는 PyPI에 게시된 경우 PyPI에서 설치
pip install lerobot_policy_my_custom_policy
```

### 정책 사용

설치가 완료되면, 여러분의 정책이 LeRobot의 학습 및 평가 도구와 자동으로 통합됩니다:

```bash
lerobot-train \
    --policy.type my_custom_policy \
    --env.type pusht \
    --steps 200000
```
