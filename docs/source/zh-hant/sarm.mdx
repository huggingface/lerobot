# SARM：階段感知獎勵建模（Stage-Aware Reward Modeling）

SARM（Stage-Aware Reward Modeling，階段感知獎勵建模）是一個以影片為基礎（video-based）、用於長水平機器人操作任務（long-horizon robot manipulation tasks）的獎勵建模框架（reward modeling framework）。本指南涵蓋如何訓練（training） SARM 獎勵模型（reward models），以及選擇性地將其與獎勵對齊行為複製（Reward-Aligned Behavior Cloning，RA-BC）一起使用。

**論文**：[SARM: Stage-Aware Reward Modeling for Long Horizon Robot Manipulation](https://arxiv.org/abs/2509.25358)

## 為什麼需要獎勵模型？（why reward models?）

標準的行為複製（behavior cloning）將所有示範影格（demonstration frames）一視同仁，但真實世界的機器人資料集（real-world robot datasets）是雜亂的。它們包含猶豫（hesitations）、修正（corrections），以及品質參差的軌跡（variable-quality trajectories）。獎勵模型（reward models）透過從示範（demonstrations）中學習一個可泛化（generalizable）的**任務進度（task progress）**概念來解決此問題：給定影片影格（video frames）與任務描述（task description），它們會預測機器人距離完成任務有多近（predict how close the robot is to completing the task, 0→1）。這個學到的「進度訊號（progress signal）」可以有多種用法，兩個有前景的應用是：（1）**加權模仿學習（imitation learning, RA-BC）**，其中高進度影格（high-progress frames）在策略訓練（policy training）期間獲得更多權重（weight）；以及（2）**強化學習（reinforcement learning, RL）**，其中獎勵模型（reward model）為線上或離線的策略改進（online or offline policy）提供密集獎勵（dense rewards）。

## 概觀（overview）

SARM 具有以下特點（features）：

1. **階段感知架構（stage-aware architecture）**：聯合預測（jointly predicts）高階任務階段（high-level task stage）以及各階段內的細粒度進度（fine-grained progress）
2. **子任務標註（subtask annotations）**：使用自然語言子任務標註（natural language subtask annotations）來推導一致的進度標籤（consistent progress labels）
3. **時間比例（temporal proportions）**：為每個子任務（subtask）計算資料集層級的先驗（dataset-level priors, α̅\_k），以在不同長度的示範間正規化進度（normalize progress across variable-length demonstrations）

SARM 針對每個影格（frame）以一個精簡的 **stage+tau** 目標（target）進行訓練（training）：

- **stage**：整數階段索引（integer stage index） `k ∈ {0, ..., K-1}`
- **τ (tau)**：階段內進度（within-stage progress） `τ ∈ [0, 1]`
- **目標編碼**：`y = k + τ`（這是資料集處理器 dataset processor 產生的內容）

在推論（inference）時（以及在下游的 RA-BC 中），SARM 會使用資料集層級（dataset-level）的**時間比例（temporal proportions）** `α̅_k`（儲存於 `meta/temporal_proportions_*.json`）將原始的 `k + τ` 值轉換為 `[0, 1]` 之間的**正規化進度（normalized progress）**。

這符合論文中的**公式 (2)**：

```
progress_t = P_{k-1} + α̅_k × τ_t
```

其中：

- `τ_t = (t - s_k) / (e_k - s_k)` 是子任務內的正規化時間（within-subtask normalized time）
- `P_{k-1}` 是累積先驗（cumulative prior，先前子任務 subtask 比例之總和）
- `α̅_k` 是子任務（subtask） k 的時間比例（temporal proportion）

這確保了相同的任務狀態（identical task states）會對映到一致的進度值（progress values），即使跨不同長度的示範（demonstrations of different lengths）也是如此。

## 輸入與目標（inputs and targets, what the new code expects）

SARM 透過其處理器（`src/lerobot/rewards/sarm/processor_sarm.py`）進行訓練（training），該處理器（processor）會：

- 用 CLIP（ViT-B/32）將影像（images）與任務文字（task text）**編碼（encode）**為 `video_features` 與 `text_features`
- 將機器人狀態（robot state）**填充/截斷（pads/truncates）**成 `state_features`（最多 `max_state_dim`）
- 使用 stage+tau 編碼（encoding） `y = k + τ` 將**目標建構（builds targets）**為 `sparse_targets`（在 `dense_only`/`dual` 中還有 `dense_targets`）
- 使用每個樣本（per-sample）的 `lengths` 張量來**遮罩倒帶影格（masks rewind frames）**（倒帶 rewind，是一種訓練時的增強 training-time augmentation）

至少，每個訓練樣本（training sample）需要：

- `task`（字串）：任務描述（task description）
- 來自資料集（dataset）的 `policy.image_key` 影像（images）與 `policy.state_key` 狀態（state）

## 步驟 5（可選）：以 RA-BC 訓練策略（train policy with RA-BC）

獎勵對齊行為複製（Reward-Aligned Behavior Cloning, RA-BC）使用訓練好的 SARM 模型（trained SARM model），根據預測的進度（predicted progress）改善來為訓練樣本加權（weight training samples）。這需要兩個步驟：

1. 使用訓練好的 SARM 模型（trained SARM model）**預先計算（precompute）**所有影格（frames）的**進度值（progress values）**
2. 使用預先計算的值（precomputed values）以 RA-BC 加權（weighting）**訓練策略（train a policy）**

### RA-BC 如何運作（how RA-BC works）

對每個訓練樣本（training sample），RA-BC 會計算進度差（progress delta）：

```
r_i = φ(o_{t+Δ}) - φ(o_t)
```

其中 `φ` 是 SARM 進度預測（progress prediction），`Δ` 是策略的（policy’s） `chunk_size`。具有正進度（positive progress，好的示範）的樣本（samples）獲得較高權重（higher weights），而具有負進度或零進度（negative or zero progress）的樣本則被降低權重（down-weighted）。

此加權（weighting）遵循論文中的**方程式 8-9**：

- **軟權重**：`w̃_i = clip((r_i − (μ − 2σ)) / (4σ + ε), 0, 1)`
- **最終權重**：`w_i = 𝟙{r_i > κ} + 𝟙{0 ≤ r_i ≤ κ} × w̃_i`

### 步驟 5a：計算 SARM 進度值（compute sarm progress values）

首先，在您資料集（dataset）中的所有影格（frames）上執行 SARM 模型（model）以計算進度值（progress values）：

```bash
python -m lerobot.rewards.sarm.compute_rabc_weights \
  --dataset-repo-id your-username/your-dataset \
  --reward-model-path your-username/sarm-model \
  --head-mode sparse \
  --num-visualizations 5 \
  --push-to-hub
```

此腳本（script）會：

- 處理所有影格（frames ）並計算進度值（progress values）
- 將進度值（progress values）儲存到磁碟上資料集（dataset）旁的一個 parquet 檔案（預設為 `<dataset_root>/sarm_progress.parquet`）
- 產生前 N 個回合（episodes）的視覺化（預設：5）

**引數（arguments）：**

| 引數（arguments）      | 描述（description）                                              | 預設     |
| ---------------------- | ---------------------------------------------------------------- | -------- |
| `--reward-model-path`  | 訓練好的 SARM 模型（trained SARM model）路徑（path）             | （必填） |
| `--head-mode`          | 要使用的 SARM 頭（head）：`sparse`、`dense` 或 `both`            | `sparse` |
| `--device`             | 推論（inference）所用的裝置（device）                            | `cuda`   |
| `--visualize-only`     | 只視覺化預測（predictions，不計算 RA-BC）                        | `false`  |
| `--num-visualizations` | 要視覺化的回合數量（number of episodes，預設：5，設為 0 以跳過） | `5`      |

**輸出格式**（`sarm_progress.parquet`）：

| 欄位（column）    | 描述（description）                                      |
| ----------------- | -------------------------------------------------------- |
| `index`           | 資料集（dataset）中的全域影格索引（global frame index）  |
| `episode_index`   | 回合編號（episode number）                               |
| `frame_index`     | 回合內的區域影格索引（local frame index within episode） |
| `progress_sparse` | sparse 頭（head）進度值 [0, 1]                           |
| `progress_dense`  | dense 頭（head）進度值 [0, 1]（若有計算）                |

### 步驟 5b：以 RA-BC 訓練策略（train policy with ra-bc）

一旦有了進度檔案（progress file），就以 RA-BC 加權訓練您的策略（train your policy）。若未明確提供，進度檔案會從資料集路徑（dataset path）自動偵測（`sarm_progress.parquet`）。目前 PI0、PI0.5 與 SmolVLA 支援 RA-BC：

```bash
lerobot-train \
  --dataset.repo_id=your-username/your-dataset \
  --policy.type=pi0 \
  --sample_weighting.type=rabc \
  --sample_weighting.head_mode=sparse \
  --sample_weighting.kappa=0.01 \
  --output_dir=outputs/train/policy_rabc \
  --batch_size=32 \
  --steps=40000
```

訓練腳本（training script）會自動：

- 從 parquet 檔案載入預先計算的進度值（precomputed progress values）
- 使用策略的（policy’s ） `chunk_size` 計算進度差（Δ）
- 根據進度（progress）改善計算樣本權重（computes sample weights）
- 在訓練（training）期間套用加權損失（weighted loss）

**RA-BC 引數（RA-BC Arguments）：**

| 引數（argument）                   | 描述（description）                                                   | 預設（default）         |
| ---------------------------------- | --------------------------------------------------------------------- | ----------------------- |
| `--sample_weighting.type`          | 加權策略類型（weighting strategy type，`rabc` 或 `uniform`）          | `rabc`                  |
| `--sample_weighting.progress_path` | 進度 parquet 檔案的路徑（path to progress parquet file）              | `sarm_progress.parquet` |
| `--sample_weighting.head_mode`     | 要使用哪個 SARM 頭的進度（SARM head’s progress）：`sparse` 或 `dense` | `sparse`                |
| `--sample_weighting.kappa`         | 高品質樣本（high-quality samples）的閾值 κ                            | `0.01`                  |
| `--sample_weighting.epsilon`       | 數值穩定性（numerical stability）的微小常數                           | `1e-6`                  |

### 調校 RA-BC 的 Kappa（tuning RA-BC Kappa）

`kappa` 參數（parameter）是決定哪些樣本（samples）獲得完整權重（w=1）的閾值（threshold）。理解如何調校它對於 RA-BC 有效運作至關重要。

**加權如何運作（how the weighting works）：**

| 條件（condition）   | 權重（weight）                     |
| ------------------- | ---------------------------------- |
| `delta > kappa`     | 1.0（硬門檻）                      |
| `0 ≤ delta ≤ kappa` | 來自 Eq. 8 的軟權重（soft weight） |
| `delta < 0`         | 0.0（負進度 negative progress）    |

**診斷 kappa 問題（diagnosing kappa issues）：**

在訓練（training）期間監控這些 WandB 指標（metrics）：

| 指標（metric）                | 健康範圍（healthy range） | 問題指標（problem indicator） |
| ----------------------------- | ------------------------- | ----------------------------- |
| `sample_weight_mean_weight`   | 0.3 - 0.8                 | ≈ 1.0 表示 kappa 太低         |
| `sample_weighting/delta_mean` | > 0                       | 應為正值                      |
| `sample_weighting/delta_std`  | > 0                       | 資料品質的變異                |

**若 `sample_weight_mean_weight ≈ 1.0`：** 您的 kappa 太低。大多數樣本的 `delta > kappa`，完全繞過了軟加權（soft-weighting）。RA-BC 便等同於普通的 BC。

**根據您的資料設定 kappa（setting kappa based on your data）：**

預設的 `kappa=0.01` 是針對論文的 T 恤摺疊任務（30fps 下約 90 秒的回合 episode）調校的。對於您的資料集（dataset），請檢查記錄的 `sample_weighting/delta_mean` 與 `sample_weighting/delta_std`：

```
# 若 delta_mean ≈ 0.03 且 delta_std ≈ 0.02：
# 大多數 delta 落在 [0.01, 0.05] 範圍內

# 選項 1：設定 kappa = delta_mean（中等選擇性）
--sample_weighting.kappa=0.03

# 選項 2：設定 kappa = delta_mean + delta_std（高選擇性）
--sample_weighting.kappa=0.05

# 選項 3：設定 kappa = delta_mean + 2*delta_std（非常有選擇性）
--sample_weighting.kappa=0.07
```

**RA-BC 何時可能沒有幫助（when RA-BC may not help）：**

若您的資料集（dataset）已經是高品質（所有示範的進度皆一致），RA-BC 不會提供太多益處，因為沒有東西可以過濾。

## 提示與最佳實務（tips & best practices）

### 選擇模式（choosing a mode）

- **從 `single_stage` 開始**進行快速實驗 - 沒有標註開銷（annotation overhead）
- 當您想要詳細的進度追蹤（progress tracking）但任務（task）沒有明確的高階階段（high-level stages）時，使用 **`dense_only`**
- 對於粗粒度與細粒度進度（coarse and fine-grained progress）皆有意義的複雜任務（complex tasks），使用 **`dual`**

### 標註品質（annotation quality）

1. **子任務名稱要具體（be specific with subtask names）**：不要用「fold」，改用「grab near side and fold toward center」
2. **以視覺化驗證（verify with visualization）**：訓練（training）前務必檢查幾個回合（few episodes）
3. **命名一致（consistent naming）**：在所有回合（episode）中使用相同的子任務名稱（same subtask names）

### RA-BC

1. **先訓練SARM（train SARM first）**：RA-BC 的品質完全取決於 SARM 的品質
2. **監控（monitor） `sample_weight_mean_weight`**：若它 ≈ 1.0，請提高 kappa（請參閱[調校 RA-BC 的 Kappa](#%E8%AA%BF%E6%A0%A1-ra-bc-%E7%9A%84-kappatuning-ra-bc-kappa)）
