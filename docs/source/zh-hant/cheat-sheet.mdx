# 小抄（Cheat sheet）

所有 LeRobot 指令集中在此處。如果您忘記了如何使用某個特定指令，或想了解一個新指令，都可以在這裡查閱。

> [!WARNING]
> 對於以下列出的所有指令，請記得將連接埠（ports）/名稱（names）/ID（id）替換為您自己的值！

> [!TIP]
> 另一個查看所有指令並針對您的特定設置進行配置的好方法，是使用這個 [Jupyter Notebook](https://github.com/huggingface/lerobot/blob/main/examples/notebooks/quickstart.ipynb)。

### 設定與安裝（setup and installation）

安裝方式請參閱 [LeRobot 安裝指南](https://huggingface.co/docs/lerobot/main/en/installation)。

### 實用工具（useful tools）

###### 尋找連接埠（find port）

使用此工具來識別您的機器人連接到哪個序列埠（serial port）。請依照終端機（terminal）中的指示操作：系統會要求您拔除 USB 線並按下 Enter。腳本接著會偵測並印出該機器人的正確序列埠。

```bash
lerobot-find-port
```

###### 尋找攝影機（find cameras）

快速找到攝影機索引並驗證其輸出。此指令會將攝影機資訊印到終端機（terminal），並將每個偵測到的攝影機的測試畫面儲存到 `lerobot/outputs/captured_images`

```bash
lerobot-find-cameras
```

### 校正（calibration）

在大多數情況下，您只需要對每個機器人和遙操作裝置（teleoperation device）執行一次校正（calibration）。在執行校正之前，請確保所有關節（joints）大致處於中間位置（middle position）。

```bash
lerobot-calibrate \
    --robot.type=so101_follower \
    --robot.port=/dev/ttyACM0 \
    --robot.id=my_follower_arm
```

請確保在後續其他腳本中使用與校正（calibration）時相同的 ID。LeRobot 就是透過這些 ID 來找到校正檔案的。

### 遙操作（teleoperation）

使用兩個攝影機進行遙操作，並透過 Rerun 顯示資料。

```bash
lerobot-teleoperate \
    --robot.type=so101_follower \
    --robot.port=/dev/ttyACM0 \
    --robot.id=my_follower_arm \
    --robot.cameras="{ top: {type: opencv, index_or_path: 1, width: 640, height: 480, fps: 30}, wrist: {type: opencv, index_or_path: 0, width: 640, height: 480, fps: 30} }" \
    --teleop.type=so101_leader \
    --teleop.port=/dev/ttyACM1 \
    --teleop.id=my_leader_arm \
    --display_data=true
```

### 錄製資料集（recording a dataset）

資料集（dataset）會自動上傳到伺服器（server）並儲存到 repo_id 下，請確保您已透過 CLI 登入 HF 帳號：
`hf auth login`

您可以從以下位址取得令牌（token）：[https://huggingface.co/settings/tokens](https://huggingface.co/settings/tokens)

```bash
lerobot-record \
    --robot.type=so101_follower \
    --robot.port=/dev/ttyACM0 \
    --robot.id=my_follower_arm \
    --robot.cameras="{ top: {type: opencv, index_or_path: 1, width: 640, height: 480, fps: 30}, wrist: {type: opencv, index_or_path: 0, width: 640, height: 480, fps: 30} }" \
    --teleop.type=so101_leader \
    --teleop.port=/dev/ttyACM1 \
    --teleop.id=my_leader_arm \
    --dataset.repo_id=${HF_USER}/so101_dataset_test \
    --dataset.num_episodes=30 \
    --dataset.single_task="put the red brick in a bowl" \
    --dataset.streaming_encoding=true \
    --display_data=true
```

在收集資料集（collecting the dataset）時，您可以使用鍵盤控制流程：
使用鍵盤快捷鍵控制資料錄製流程（data recording flow）：

- 按下 **右方向鍵 (`→`)**：儲存當回合（episode）並進入下一回合。
- 按下 **左方向鍵 (`←`)**：刪除當前回合（episode）並重新錄製。
- 按下 **Escape (`ESC`)**：停止錄製、編碼影片（encode videos）並上傳。

### 錄製深度（recording depth）

Intel RealSense 攝影機（`type: intelrealsense`）在您設定 `use_depth: true` 時會錄製深度串流（record a depth stream）。深度會量化為 12 位元編碼（quantized to 12-bit codes），並儲存為獨立的影片。

```bash
lerobot-record \
    ... \
    --robot.cameras="{ head: {type: intelrealsense, serial_number_or_name: \"0123456789\", width: 640, height: 480, fps: 30, use_depth: true} }" \
    --dataset.repo_id=${HF_USER}/so101_depth_test \
    --dataset.single_task="put the red brick in a bowl" \
    --dataset.depth_encoder.depth_min=0.01 \
    --dataset.depth_encoder.depth_max=10.0 \
    --dataset.depth_encoder.shift=0.0 \
    --dataset.depth_encoder.use_log=true
```

### 影片編碼參數（video encoding parameters）

RGB 與深度串流（stream）會透過 `--dataset.rgb_encoder.*` 和 `--dataset.depth_encoder.*` 這些鍵各自獨立編碼。

```bash
lerobot-record \
    ... \
    --dataset.rgb_encoder.vcodec=h264 \
    --dataset.rgb_encoder.pix_fmt=yuv420p \
    --dataset.rgb_encoder.crf=23 \
    --dataset.depth_encoder.vcodec=hevc \
    --dataset.depth_encoder.extra_options='{"x265-params": "lossless=1"}'
```

### 訓練（training）

根據您的硬體配置，訓練策略（train the policy）可能需要數小時。以下是訓練簡單 `ACT` 策略的方式：

```bash
lerobot-train \
    --dataset.repo_id=${HF_USER}/so101_dataset_test \
    --policy.type=act \
    --output_dir=outputs/train/act_so101_test \
    --job_name=act_so101_test \
    --policy.device=cuda \
    --wandb.enable=true \
    --policy.repo_id=${HF_USER}/policy_test \
    --steps=20000
```

- 策略類型（policy types）：`act`、`diffusion`、`smolvla`、`pi05`
- 裝置：`cuda`（NVIDIA）、`mps`（Apple Silicon）、`cpu`

如果您想微調特定模型（fine-tune a specific model），可以提供模型的路徑（path to the model）。在這種情況下只需提供 path，可以省略 type。

```bash
lerobot-train \
    --dataset.repo_id=${HF_USER}/so101_dataset_test \
    --policy.path=username/the_policy_to_finetune \
    --policy.device=cuda \
    --policy.repo_id=${HF_USER}/policy_test \
    --output_dir=outputs/train/act_so101_test \
    --steps=20000
```

沒有本機 GPU 嗎？在任一指令中加上 `--job.target=<flavor>`（例如 `a10g-small`），`lerobot-train` 就會改在 [Hugging Face Jobs](https://huggingface.co/docs/hub/jobs) 上執行；它會為您上傳僅存於本機的資料集（local-only dataset），並推送訓練好的模型（pushes the trained model）。使用 `hf jobs hardware` 可列出可用的 flavor。

若要接續（resume）訓練，將 `--config_path` 指向某個檢查點（checkpoint）並加上 `--resume=true`。它接受本機路徑（local path）或 Hub 儲存庫的 repo id（會取得最新的檢查點），在本機或透過加上 `--job.target=<flavor>` 在 Job 上執行皆可：

```bash
lerobot-train --config_path=${HF_USER}/policy_test --resume=true --job.target=a10g-small
```

### 推論（inference）

推論（inference）是指在機器人上運行已訓練的策略/模型（trained policy/model）。為此我們使用 `lerobot-rollout`。您需要提供策略的路徑（path to your policy），可以是本機路徑或 Hugging Face 上的路徑，例如 "lerobot/folding_latest"。您的攝影機配置（cameras configuration）需要與收集資料集（collecting the dataset）時使用的一致。持續時間以秒為單位，若未指定則會無限期運行。

> [!TIP]
> 如果您使用的是先前的 V0.5.1 版本，則需要使用 `lerobot-record` 而非 `lerobot-rollout`。更多資訊請參閱[此處](https://huggingface.co/docs/lerobot/v0.5.1/en/il_robots#run-inference-and-evaluate-your-policy)。

```bash
lerobot-rollout \
    --strategy.type=base \
    --policy.path=${HF_USER}/my_policy \
    --robot.type=so101_follower \
    --robot.port=/dev/ttyACM1 \
    --robot.cameras="{ up: {type: opencv, index_or_path: /dev/video1, width: 640, height: 480, fps: 30}, side: {type: opencv, index_or_path: /dev/video5, width: 640, height: 480, fps: 30}}" \
    --task="Put lego brick into the transparent box" \
    --duration=60
```
