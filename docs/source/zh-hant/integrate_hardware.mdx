# 自備硬體（bring your own hardware）

本教學將說明如何將你自己的機器人設計整合到 LeRobot 生態系統中，並使其能存取我們所有的工具（資料收集 data collection、控制管線 control pipelines、策略訓練 policy training 與推論 inference）。

為此，我們在 LeRobot 中提供了 [`Robot`](https://github.com/huggingface/lerobot/blob/main/src/lerobot/robots/robot.py) 基底類（base class），它定義了實體機器人整合的標準介面（standard interface）。讓我們來看看如何實作它。

## 先決條件（prerequisites）

- 您的機器人有暴露的通訊介面（communication interface ，例如序列埠 serial port、CAN、TCP）
- 能以程式化方式讀取感測器資料（sensor data）並傳送馬達指令（motor commands）的方式，例如製造商提供的 SDK 或 API，或是您自行實作的通訊協定（protocol）。
- 已在您的環境中安裝 LeRobot。請依照我們的[安裝指南](./installation)進行。

## 選擇你的馬達（choose your motors）

如果你使用的是 Feetech 或 Dynamixel 馬達，LeRobot 提供了內建的匯流排介面（bus interfaces）：

- [`FeetechMotorsBus`](https://github.com/huggingface/lerobot/blob/main/src/lerobot/motors/feetech/feetech.py) – 用於控制 Feetech 的伺服馬達
- [`DynamixelMotorsBus`](https://github.com/huggingface/lerobot/blob/main/src/lerobot/motors/dynamixel/dynamixel.py) – 用於控制 Dynamixel 的伺服馬達

請參考 [`MotorsBus`](https://github.com/huggingface/lerobot/blob/main/src/lerobot/motors/motors_bus.py) 抽象類（abstract class）來了解其 API。
要看一個好的使用範例，你可以參考我們自己的 [SO101 從臂實作](https://github.com/huggingface/lerobot/blob/main/src/lerobot/robots/so_follower/so_follower.py)

如果相容的話請使用這些介面（interface）。否則，你需要找到或撰寫一個 Python 介面（本教學不涵蓋此內容）：

- 找到現有的 Python SDK（或使用 C/C++ 的綁定）
- 或實作一個基本的通訊包裝器（communication wrapper，例如透過 pyserial、socket 或 CANopen）

你並不孤單——許多社群貢獻使用了自訂的開發板或韌體！

對於 Feetech 和 Dynamixel，我們目前支援以下伺服馬達：- Feetech：- STS 與 SMS 系列（協定 0）：`sts3215`、`sts3250`、`sm8512bl` - SCS 系列（協定 1）：`scs0009` - Dynamixel（僅協定 2.0）：`xl330-m077`、`xl330-m288`、`xl430-w250`、`xm430-w350`、`xm540-w270`、`xc430-w150`

如果你使用的 Feetech 或 Dynamixel 伺服馬達不在此列表中，你可以在 [Feetech 表格](https://github.com/huggingface/lerobot/blob/main/src/lerobot/motors/feetech/tables.py)或 [Dynamixel 表格](https://github.com/huggingface/lerobot/blob/main/src/lerobot/motors/dynamixel/tables.py)中新增。根據型號不同，這需要你添加型號特定的資訊。不過在大多數情況下，需要添加的內容不會太多。

在接下來的章節中，我們將使用 `FeetechMotorsBus` 作為範例中的馬達介面（motors interface）。如有需要，請替換並適配你的馬達。

## 步驟 1: 建立 `Robot` 介面的子類（subclass the robot interface）

你首先需要為你的機器人指定其配置類（config class）和一個字串識別碼（string identifier, `name`）。如果你的機器人有特殊需求且你希望能容易變更，那些應該放在這裡（例如連接埠 port/位址address, 鮑率 baudrate）。

在這裡，我們將為我們的機器人新增連接埠名稱（port name）和一個預設的攝影機：

<!-- prettier-ignore-start -->
```python
from dataclasses import dataclass, field

from lerobot.cameras import CameraConfig
from lerobot.cameras.opencv import OpenCVCameraConfig
from lerobot.robots import RobotConfig


@RobotConfig.register_subclass("my_cool_robot")
@dataclass
class MyCoolRobotConfig(RobotConfig):
    port: str
    cameras: dict[str, CameraConfig] = field(
        default_factory={
            "cam_1": OpenCVCameraConfig(
                index_or_path=2,
                fps=30,
                width=480,
                height=640,
            ),
        }
    )
```
<!-- prettier-ignore-end -->

[攝影機教學](./cameras)以了解如何偵測並新增您的攝影機。

接下來，我們將建立繼承（inherits）自 `Robot` 的實際機器人類（actual robot class）。這個抽象類（abstract class）定義了一個規範（contract），你的機器人必須遵循它才能與 LeRobot 的其餘工具配合使用。

在這裡我們將建立一個帶有一個攝影機的簡單 5 自由度（5-DoF）機器人。它可以是一個簡單的機械臂，但請注意 `Robot` 抽象類（abstract class）不對你的機器人形態（form factor）做任何假設。設計新機器人時你可以盡情發揮想像力！

<!-- prettier-ignore-start -->
```python
from lerobot.cameras import make_cameras_from_configs
from lerobot.motors import Motor, MotorNormMode
from lerobot.motors.feetech import FeetechMotorsBus
from lerobot.robots import Robot

class MyCoolRobot(Robot):
    config_class = MyCoolRobotConfig
    name = "my_cool_robot"

    def __init__(self, config: MyCoolRobotConfig):
        super().__init__(config)
        self.bus = FeetechMotorsBus(
            port=self.config.port,
            motors={
                "joint_1": Motor(1, "sts3250", MotorNormMode.RANGE_M100_100),
                "joint_2": Motor(2, "sts3215", MotorNormMode.RANGE_M100_100),
                "joint_3": Motor(3, "sts3215", MotorNormMode.RANGE_M100_100),
                "joint_4": Motor(4, "sts3215", MotorNormMode.RANGE_M100_100),
                "joint_5": Motor(5, "sts3215", MotorNormMode.RANGE_M100_100),
            },
            calibration=self.calibration,
        )
        self.cameras = make_cameras_from_configs(config.cameras)
```
<!-- prettier-ignore-end -->

## 步驟 2：定義觀測與動作特徵（define observation and action features）

這兩個屬性（properties）定義了你的機器人與使用它的工具（如資料收集 data collection 或學習管線 learning pipeline）之間的*介面規格（interface contract）*。

> [!WARNING]
> 請注意，這些屬性（properties）必須在機器人尚未連線的情況下也能呼叫，因此避免依賴執行時的硬體狀態（runtime hardware state）來定義它們。

### `observation_features`

此屬性（property）應返回一個字典（dictionary），描述機器人感測器輸出的結構（structure）。鍵值（keys）對應 `get_observation()` 返回的內容，值（values）則描述形狀（shape，用於陣列 array /影像 image）或型別（type，用於簡單數值）。

我們的 5 自由度（5-DoF）手臂搭配一個攝影機的範例：

<!-- prettier-ignore-start -->
```python
@property
def _motors_ft(self) -> dict[str, type]:
    return {
        "joint_1.pos": float,
        "joint_2.pos": float,
        "joint_3.pos": float,
        "joint_4.pos": float,
        "joint_5.pos": float,
    }

@property
def _cameras_ft(self) -> dict[str, tuple]:
    return {
        cam: (self.cameras[cam].height, self.cameras[cam].width, 3) for cam in self.cameras
    }

@property
def observation_features(self) -> dict:
    return {**self._motors_ft, **self._cameras_ft}
```
<!-- prettier-ignore-end -->

在這個案例中，觀測為一個儲存每個馬達的位置（motor’s position）和一張攝影機影像（camera image）的簡單字典（dict）。

### `action_features`

此屬性（property）描述你的機器人透過 send_action() 期望接收的指令。同樣，鍵值（keys）必須符合預期的輸入格式，值（values）則定義每個指令的形狀（shape）/型別（type）。

這裡，我們簡單地使用與 `observation_features` 相同的關節本體感覺（joints proprioceptive features, `self._motors_ft`）：發送的動作（action）就是每個馬達的目標位置（goal position）。

<!-- prettier-ignore-start -->
```python
def action_features(self) -> dict:
    return self._motors_ft
```
<!-- prettier-ignore-end -->

## 步驟 3：處理連線與斷線（handle connection and disconnection）

這些方法（methods）應處理與你的硬體（例如序列埠 serial port、CAN 介面、USB 裝置、攝影機）的通訊（communication）開啟與關閉。

### `is_connected`

此屬性（property）應簡單地反映是否已與機器人硬體建立通訊（communication）。當此屬性為 `True` 時，應該能夠使用 `get_observation()` 和 `send_action()` 對硬體進行讀寫（read and write）。

<!-- prettier-ignore-start -->
```python
@property
def is_connected(self) -> bool:
    return self.bus.is_connected and all(cam.is_connected for cam in self.cameras.values())
```
<!-- prettier-ignore-end -->

### `connect()`

此方法（method）應建立與硬體的通訊（communication）。此外，如果你的機器人需要校正（calibration）且尚未校正，預設應啟動校正程序。如果你的機器人需要特定配置（specific configuration），也應在此呼叫。

<!-- prettier-ignore-start -->
```python
def connect(self, calibrate: bool = True) -> None:
    self.bus.connect()
    if not self.is_calibrated and calibrate:
        self.calibrate()

    for cam in self.cameras.values():
        cam.connect()

    self.configure()
```
<!-- prettier-ignore-end -->

### `disconnect()`

此方法應優雅地終止與硬體的通訊（communication）：釋放任何相關資源（執行緒 thread 或程序 processes）、關閉連接埠（ports）等。

在這裡，我們已經在 `MotorsBus` 和 `Camera` 類（classes）中處理了這些，所以只需要呼叫它們自己的 `disconnect()` 方法（methods）：

<!-- prettier-ignore-start -->
```python
def disconnect(self) -> None:
    self.bus.disconnect()
    for cam in self.cameras.values():
        cam.disconnect()
```
<!-- prettier-ignore-end -->

## 步驟 4：支援校正與配置設定（support calibration and configuration）

LeRobot 支援自動儲存和載入校正資料（calibration data）。這對於關節偏移量（joint offsets）、零位位置或感測器對齊（sensor alignment）非常有用。

> 請注意，根據你的硬體不同，這可能不適用。如果是這種情況，你可以簡單地將這些方法留為空操作（no-ops）：

<!-- prettier-ignore-start -->
```python
@property
def is_calibrated(self) -> bool:
    return True

def calibrate(self) -> None:
    pass
```
<!-- prettier-ignore-end -->

### `is_calibrated`

這應反映你的機器人是否已載入所需的校正（calibration）。

<!-- prettier-ignore-start -->
```python
@property
def is_calibrated(self) -> bool:
    return self.bus.is_calibrated
```
<!-- prettier-ignore-end -->

### `calibrate()`

校正（calibration）的目標有兩個：

- 了解每個馬達的物理運動範圍（physical range of motion），以便僅在此範圍內發送指令。
- 將原始馬達位置（raw motors positions）正規化（normalization）為有意義的連續值（continuous values，例如百分比、角度），而非取決於特定馬達的任意離散值（discrete value），這些值在其他地方無法複製（not replicate elsewhere）。

它應實作校正的邏輯（logic for calibration，如果適用）並更新 `self.calibration` 字典。如果你使用的是 Feetech 或 Dynamixel 馬達，我們的匯流排介面（bus interface）已包含協助此操作的方法（methods）。

<!-- prettier-ignore-start -->
```python
def calibrate(self) -> None:
    self.bus.disable_torque()
    for motor in self.bus.motors:
        self.bus.write("Operating_Mode", motor, OperatingMode.POSITION.value)

    input(f"Move {self} to the middle of its range of motion and press ENTER....")
    homing_offsets = self.bus.set_half_turn_homings()

    print(
        "Move all joints sequentially through their entire ranges "
        "of motion.\nRecording positions. Press ENTER to stop..."
    )
    range_mins, range_maxes = self.bus.record_ranges_of_motion()

    self.calibration = {}
    for motor, m in self.bus.motors.items():
        self.calibration[motor] = MotorCalibration(
            id=m.id,
            drive_mode=0,
            homing_offset=homing_offsets[motor],
            range_min=range_mins[motor],
            range_max=range_maxes[motor],
        )

    self.bus.write_calibration(self.calibration)
    self._save_calibration()
    print("Calibration saved to", self.calibration_fpath)
```
<!-- prettier-ignore-end -->

### `configure()`

使用此方法為你的硬體設定任何配置（configuration，伺服馬達控制模式、控制器增益等）。這通常應在連線時執行且具有冪等性（idempotent）。

<!-- prettier-ignore-start -->
```python
def configure(self) -> None:
    with self.bus.torque_disabled():
        self.bus.configure_motors()
        for motor in self.bus.motors:
            self.bus.write("Operating_Mode", motor, OperatingMode.POSITION.value)
            self.bus.write("P_Coefficient", motor, 16)
            self.bus.write("I_Coefficient", motor, 0)
            self.bus.write("D_Coefficient", motor, 32)
```
<!-- prettier-ignore-end -->

## 步驟 5：實作感測器讀取與動作發送（implement sensors reading and action sending）

這些是最重要的執行時函數（runtime functions）：核心 I/O 迴圈（core I/O loop）。

### `get_observation()`

從機器人返回一個感測器數值的字典（dictionary）。這些通常包括馬達狀態（motor states）、攝影機畫面（camera frames）、各種感測器等。在 LeRobot 框架（framework）中，這些觀測（observations）將被饋送給策略（policy）以預測（predict）要採取的動作（actions to take）。字典的鍵（dictionary keys）和結構（structure）必須與 `observation_features` 相符。

<!-- prettier-ignore-start -->
```python
def get_observation(self) -> dict[str, Any]:
    if not self.is_connected:
        raise ConnectionError(f"{self} is not connected.")

    # Read arm position
    obs_dict = self.bus.sync_read("Present_Position")
    obs_dict = {f"{motor}.pos": val for motor, val in obs_dict.items()}

    # Capture images from cameras
    for cam_key, cam in self.cameras.items():
        obs_dict[cam_key] = cam.async_read()

    return obs_dict
```
<!-- prettier-ignore-end -->

### `send_action()`

接收一個與 `action_features` 相符的字典（dictionary），並將其發送到你的硬體。你可以加入安全限制（safety limits，截斷、平滑），並返回實際發送的內容。

為了簡化，在我們的範例中不會對動作（action）做任何修改。

<!-- prettier-ignore-start -->
```python
def send_action(self, action: dict[str, Any]) -> dict[str, Any]:
    goal_pos = {key.removesuffix(".pos"): val for key, val in action.items()}

    # Send goal position to the arm
    self.bus.sync_write("Goal_Position", goal_pos)

    return action
```
<!-- prettier-ignore-end -->

## 新增遙操作器（adding a teleoperator）

對於實作遙操作裝置（teleoperation devices），我們也提供了 [`Teleoperator`](https://github.com/huggingface/lerobot/blob/main/src/lerobot/teleoperators/teleoperator.py) 基底類（base class）。此類與 `Robot` 基底類非常相似，同樣不對外形因素（form factor）做任何假設。

主要差異在於 I/O 函式：遙操作器（teleoperator）允許你透過 `get_action` 產生動作（action），並可以透過 `send_feedback` 接收回饋動作（feedback actions）。回饋（feedback）可以是遙操作裝置（teleoperation device）上任何可控制的東西，幫助操控者理解所發送動作的後果。例如主臂上的運動（motion）/力回饋（force）、遊戲手把控制器上的振動等。要實作遙操作器（teleoperator），你可以遵循本教學並針對這兩個方法進行調整。

## 使用你自己的 `LeRobot` 裝置 🔌（using your own Lerobot devices）

你可以透過建立一個獨立的、可安裝的 Python 套件，輕鬆地以你的自訂硬體擴展 `lerobot`——無論是攝影機、機器人還是遙操作（teleoperation）裝置。只要你遵循一些簡單的慣例，`lerobot` 的命令列工具（如 `lerobot-teleop` 和 `lerobot-record`）將**自動發現並整合你的裝置**，而無需對 `lerobot` 原始碼做任何修改。

本指南概述了你的外掛（plugin）必須遵循的慣例（conventions）。

### 4 項核心慣例（4 core conventions）

為確保你的自訂裝置（custom device）可被發現，你必須遵循以下四條規則。

#### 1\. 建立具有特定前綴的可安裝套件（create an installable package with a specific prefix）

您的專案必須是一個標準、可安裝的 Python 套件（package）。至關重要的是，您套件的名稱（定義於 `pyproject.toml` 或 `setup.py` 中）必須以下列前綴（prefixes）之一開頭：

- `lerobot_robot_` 用於機器人。
- `lerobot_camera_` 用於攝影機。
- `lerobot_teleoperator_` 用於遙操作裝置（teleoperation device）。

這個前綴系統是 `lerobot` 在 Python 環境中自動找到你的外掛的方式。

#### 2\. 遵循 `SomethingConfig`/`Something` 的命名模式（follow the somethingconfig / something naming pattern）

你的裝置實作類（device’s implementation class）必須以其設定類（configuration class）命名，只需移除 `Config` 後綴（suffix）。

- **設定類（config class）：** `MyAwesomeTeleopConfig`
- **裝置類（device class）：** `MyAwesomeTeleop`

#### 3\. 將你的檔案放在可預測的結構中（place your files in a predictable structure）

裝置類（device class, `MyAwesomeTeleop`）必須位於相對於其配置類（configuration class, `MyAwesomeTeleopConfig`）的可預測模組（predictable module）中。`lerobot` 將自動在以下位置搜尋：

- 與設定類（config class）**相同的模組（same module）**中。
- 在一個**以裝置命名的子模組（submodule named after the device）**中（例如 `my_awesome_teleop.py`）。

建議且最簡單的結構是將它們放在同一目錄中清楚命名的獨立檔案中。

#### 4\. 在 `__init__.py` 中暴露類（expose classes `in __init__.py`）

你的套件的 `__init__.py` 檔案應導入並暴露設定類（configuration class）和裝置類（device classes），使它們容易被存取。

### 全部串起來：完整範例（a complete example）

讓我們建立一個名為 `my_awesome_teleop` 的新遙操作器（teleoperation）。

#### 目錄結構（directory structure）

以下是專案資料夾的結構。套件名稱 `lerobot_teleoperator_my_awesome_teleop` 遵循**慣例 #1**。

```
lerobot_teleoperator_my_awesome_teleop/
├── pyproject.toml #（或 setup.py）列出 lerobot 作為依賴
└── lerobot_teleoperator_my_awesome_teleop/
    ├── __init__.py
    ├── config_my_awesome_teleop.py
    └── my_awesome_teleop.py
```

#### 檔案內容（file contents）

- **`config_my_awesome_teleop.py`**：定義配置類（configuration class）。注意 `Config` 後綴（suffix，**慣例 #2**）。

  ```python
  from dataclasses import dataclass

  from lerobot.teleoperators.config import TeleoperatorConfig

  @TeleoperatorConfig.register_subclass("my_awesome_teleop")
  @dataclass
  class MyAwesomeTeleopConfig(TeleoperatorConfig):
      # 你的配置放在這裡
      port: str = "192.168.1.1"
  ```

- **`my_awesome_teleop.py`**：實作裝置（device）。類名稱（class name） `MyAwesomeTeleop` 與其配置類的別名稱相符（config class name，**慣例 #2**）。此檔案結構遵循**慣例 #3**。

  ```python
  from lerobot.teleoperators.teleoperator import Teleoperator

  from .config_my_awesome_teleop import MyAwesomeTeleopConfig

  class MyAwesomeTeleop(Teleoperator):
      config_class = MyAwesomeTeleopConfig
      name = "my_awesome_teleop"

      def __init__(self, config: MyAwesomeTeleopConfig):
          super().__init__(config)
          self.config = config

      # 你的裝置邏輯（例如 connect）放在這裡
  ```

- **`__init__.py`**：暴露關鍵類別（key classes，**慣例 #4**）。

  ```python
  from .config_my_awesome_teleop import MyAwesomeTeleopConfig
  from .my_awesome_teleop import MyAwesomeTeleop
  ```

### 安裝與使用（installation and usage）

1. **在你的 Python 環境中安裝你的新外掛（plugin）。** 你可以使用 `pip` 的可編輯模式（editable mode）或從 PyPi 安裝你的本地外掛程式套件（local plugin package）。

   ```bash
   # 本地安裝
   # 導航到你的外掛程式根目錄並安裝
   ```
