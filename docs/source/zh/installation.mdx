# 安裝

本指南使用 `conda`（透過 miniforge）來管理環境（推薦）。若您偏好其他環境管理工具（例如 `uv`、`venv`），請確保您使用 Python >=3.12 以及包含 `libsvtav1` 編碼器的 `ffmpeg`，然後直接跳至[環境設定](#step-2-environment-setup)。

## 步驟 1（僅限 conda）：安裝 [`miniforge`](https://conda-forge.org/download/)

```bash
wget "https://github.com/conda-forge/miniforge/releases/latest/download/Miniforge3-$(uname)-$(uname -m).sh"
bash Miniforge3-$(uname)-$(uname -m).sh
```

## 步驟 2：環境設定

建立 Python 3.12 的虛擬環境：

<!-- prettier-ignore-start -->
<hfoptions id="create_venv">
<hfoption id="conda">
```bash
conda create -y -n lerobot python=3.12
```
</hfoption>
<hfoption id="uv">
```bash
uv python install 3.12
uv venv --python 3.12
```
</hfoption>
</hfoptions>
<!-- prettier-ignore-end -->

接著啟動虛擬環境，每次開啟新的 shell 要使用 lerobot 時都需要執行此步驟：

<!-- prettier-ignore-start -->
<hfoptions id="activate_venv">
<hfoption id="conda">```bash
conda activate lerobot
```</hfoption>
<hfoption id="uv">
```bash
# Linux/macOS
source .venv/bin/activate
# Windows PowerShell
source .venv\Scripts\Activate.ps1
```
</hfoption>
</hfoptions>
<!-- prettier-ignore-end -->

使用 `conda` 時，需在環境中安裝 `ffmpeg`：

```bash
conda install ffmpeg -c conda-forge
ffmpeg -version  # 尚不支援 ffmpeg 8.X！
```

> [!TIP]
> 通常此指令會為您的平台安裝透過 `libsvtav1` 編碼器編譯的 `ffmpeg 7.X`。若 `libsvtav1` 不支援（可透過 `ffmpeg -encoders` 查看支援的編碼器），您可以：
>
> - _[適用所有平台]_ 使用以下指令明確安裝 `ffmpeg 7.X` ：
>
> ```bash
> conda install ffmpeg=7.1.1 -c conda-forge
> ```
>
> - _[僅限 Linux]_ 若您想自行提供 ffmpeg：請安裝 [ffmpeg 建置相依套件](https://trac.ffmpeg.org/wiki/CompilationGuide/Ubuntu#GettheDependencies)並[從原始碼以 libsvtav1 編譯 ffmpeg](https://trac.ffmpeg.org/wiki/CompilationGuide/Ubuntu#libsvtav1)，並確保使用 `which ffmpeg` 確認您安裝所對應的 ffmpeg 二進位檔。

> [!NOTE]
> 在 WSL（Windows Subsystem for Linux）中安裝 LeRobot 時，請務必使用以下指令安裝 `evdev`：
>
> ```bash
> conda install evdev -c conda-forge
> ```

> [!IMPORTANT]
> 若您使用 `uv`，則必須在系統層級（虛擬環境外部）安裝 `ffmpeg`。您需要依賴 `uv` 與 `torchcodec` 動態連結至系統的 `ffmpeg`。

## 步驟 3：安裝 LeRobot 🤗

### 以原始碼安裝

首先，複製儲存庫並進入該目錄：

```bash
git clone https://github.com/huggingface/lerobot.git
cd lerobot
```

接著，以可編輯模式安裝此函式庫。如果您打算對程式碼做出貢獻，這種方式會非常有幫助。

<!-- prettier-ignore-start -->
<hfoptions id="install_lerobot_src">
<hfoption id="conda">
```bash
pip install -e .
```
</hfoption>
<hfoption id="uv">
```bash
uv pip install -e .
```
</hfoption>
</hfoptions>
<!-- prettier-ignore-end -->

### 從 PyPI 安裝

**核心函式庫：**
使用以下指令安裝基本套件：

<!-- prettier-ignore-start -->
<hfoptions id="install_lerobot_pypi">
<hfoption id="conda">
```bash
pip install lerobot
```
</hfoption>
<hfoption id="uv">
```bash
uv pip install lerobot
```
</hfoption>
</hfoptions>
<!-- prettier-ignore-end -->

_此方式僅安裝預設相依套件。_

**額外功能：**
若要安裝額外功能，請使用以下其中一項指令（若您使用 `uv`，請將下列指令中的 `pip install` 替換為 `uv pip install`）：

```bash
pip install 'lerobot[all]'          # 所有可用功能
pip install 'lerobot[aloha,pusht]'  # 指定功能（Aloha 與 Pusht）
pip install 'lerobot[feetech]'      # Feetech 馬達支援
```

_將 `[...]` 替換為您所需的功能。_

**可用標籤：**
如需完整的選用相依套件清單，請參閱：
https://pypi.org/project/lerobot/

### 疑難排解

若您遇到建置錯誤，可能需要安裝額外相依套件：`cmake`、`build-essential` 以及 `ffmpeg libs`。
若要在 Linux 上安裝這些套件，請執行：

```bash
sudo apt-get install cmake build-essential python3-dev pkg-config libavformat-dev libavcodec-dev libavdevice-dev libavutil-dev libswscale-dev libswresample-dev libavfilter-dev
```

對於其他作業系統請參閱：[編譯 PyAV](https://pyav.org/docs/develop/overview/installation.html#bring-your-own-ffmpeg)

## 選用的相依套件

LeRobot 為特定功能提供可選用的擴充套件。多個擴充套件可以組合使用（例如 `.[aloha,feetech]`）。如需所有可用的擴充套件，請參閱 `pyproject.toml`。若您使用 `uv`，請將下列指令中的 `pip install` 替換為 `uv pip install`。

### 模擬環境

安裝環境套件：`aloha`（[gym-aloha](https://github.com/huggingface/gym-aloha)）或 `pusht`（[gym-pusht](https://github.com/huggingface/gym-pusht)）
範例：

```bash
pip install -e ".[aloha]" # 或例如 "[pusht]"
```

### 馬達控制

針對 Koch v1.1 請安裝 Dynamixel SDK，對於 SO100/SO101/Moss 則安裝 Feetech SDK。

```bash
pip install -e ".[feetech]" # 或例如 "[dynamixel]"
```

### 實驗追蹤

若要使用 [Weights and Biases](https://docs.wandb.ai/quickstart) 進行實驗追蹤，請使用以下指令登入：

```bash
wandb login
```

如果你的機器人尚未組裝完成，現在可以開始組裝了。請在左側找到你的機器人型號，然後點擊連結，搭配你的機器人使用 LeRobot。
