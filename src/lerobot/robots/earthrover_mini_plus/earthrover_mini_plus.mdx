# EarthRover Mini Plus

The EarthRover Mini Plus is a cloud-connected mobile robot that uses the Frodobots SDK for control and telemetry. This integration allows you to record datasets and train policies using LeRobot's imitation learning framework.

## Prerequisites

### Hardware Requirements

- EarthRover Mini robot with Frodobots SDK support
- Laptop/PC with Python 3.10+
- Network connection to access the Frodobots SDK server

### Frodobots SDK Setup

The EarthRover Mini Plus requires the [Frodobots SDK](https://github.com/Frodobots/earth-rovers-sdk) to be running. Follow these steps:

1. Clone and setup the SDK:
```bash
git clone https://github.com/Frodobots/earth-rovers-sdk.git
cd earth-rovers-sdk
pip install -r requirements.txt
```

2. Start the SDK server:
```bash
hypercorn main:app --reload
```

The SDK will run on `http://localhost:8000` by default and provide:
- Robot control commands
- Camera feeds (front and rear)
- Telemetry data (battery, orientation, speed)

> [!IMPORTANT]
> Make sure the SDK server is running before connecting your robot.

## Install LeRobot

Follow our [Installation Guide](./installation) to install LeRobot.

In addition to the base installation, install the EarthRover Mini dependencies:

```bash
pip install -e .
```

## Robot Configuration

The EarthRover Mini Plus uses a cloud-based architecture where:
- **Robot Control**: HTTP POST requests to `/control` endpoint
- **Camera Feeds**: HTTP GET requests to `/v2/front` and `/v2/rear` endpoints  
- **Telemetry**: HTTP GET requests to `/data` endpoint

No USB ports or physical connections are required - everything is handled through the SDK API.

## Calibration

The EarthRover Mini Plus does not require calibration as it's controlled through the cloud SDK. The robot is ready to use once the SDK server is running and accessible.

## Teleoperation

To teleoperate the robot, you'll use keyboard controls. The keyboard teleoperator provides intuitive WASD controls for driving and rotation.

### Keyboard Controls

| Key | Action |
| --- | -------------- |
| W   | Move forward   |
| S   | Move backward  |
| A   | Turn left (with forward motion) |
| D   | Turn right (with forward motion) |
| Q   | Rotate left in place |
| E   | Rotate right in place |
| Space | Stop all movement |
| +/= | Increase speed |
| -   | Decrease speed |
| ESC | Disconnect |

### Speed Modes

The robot supports adjustable speeds:
- **Linear speed**: Configurable (default: 50 units)
- **Angular speed**: Configurable (default: 30 units)
- **Speed increment**: Adjustable via +/- keys (default: 10 units)

### Test Teleoperation

You can test teleoperation without recording by running:

```python
from lerobot.robots.earthrover_mini_plus import EarthRoverMiniPlus, EarthRoverMiniPlusConfig
from lerobot.teleoperators.keyboard import KeyboardRoverTeleop, KeyboardRoverTeleopConfig

# Initialize robot
robot_config = EarthRoverMiniPlusConfig()
robot = EarthRoverMiniPlus(robot_config)

# Initialize teleoperator
teleop_config = KeyboardRoverTeleopConfig(
    linear_speed=50.0,
    angular_speed=30.0,
    speed_increment=10.0
)
teleop = KeyboardRoverTeleop(teleop_config)

# Connect
robot.connect()
teleop.connect()

# Teleoperate (use keyboard controls)
try:
    while True:
        action = teleop.get_action()
        robot.send_action(action)
except KeyboardInterrupt:
    pass
finally:
    robot.disconnect()
    teleop.disconnect()
```

> [!TIP]
> If you're using a Mac, you might need to give Terminal permission to access your keyboard for teleoperation. Go to System Preferences > Security & Privacy > Input Monitoring and check the box for Terminal.

## Record a Dataset

Once you're comfortable with teleoperation, you can record your first dataset. Recording captures:
- **Actions**: Linear and angular velocities from keyboard
- **Observations**: Front and rear camera images (640x480), battery level, orientation, current velocities
- **Timestamps**: For proper temporal alignment

### Setup Hugging Face Hub

We use the Hugging Face Hub for uploading datasets. If you haven't used the Hub before, login with a write-access token from [Hugging Face settings](https://huggingface.co/settings/tokens):

```bash
huggingface-cli login --token ${HUGGINGFACE_TOKEN} --add-to-git-credential
```

Store your Hugging Face username:

```bash
HF_USER=$(huggingface-cli whoami | head -n 1)
echo $HF_USER
```

### Record Episodes

Use the provided recording script:

```bash
python examples/earthrover_mini/record.py
```

Or customize your recording parameters:

```python
from lerobot.datasets.lerobot_dataset import LeRobotDataset
from lerobot.robots.earthrover_mini_plus import EarthRoverMiniPlus, EarthRoverMiniPlusConfig
from lerobot.teleoperators.keyboard import KeyboardRoverTeleop, KeyboardRoverTeleopConfig

# Configuration
HF_REPO_ID = f"{HF_USER}/earthrover-navigation"
NUM_EPISODES = 10
FPS = 10  # Lower FPS for cloud-based robot
EPISODE_TIME_S = 30
TASK_DESCRIPTION = "Navigate around obstacles"

# Create robot and teleoperator
robot = EarthRoverMiniPlus(EarthRoverMiniPlusConfig())
teleop = KeyboardRoverTeleop(KeyboardRoverTeleopConfig())

# Create dataset
dataset = LeRobotDataset.create(
    repo_id=HF_REPO_ID,
    fps=FPS,
    robot_type=robot.name,
    use_videos=True,
)

# Connect and record
robot.connect()
teleop.connect()

# Record episodes...
# (See examples/earthrover_mini/record.py for full implementation)
```

### Dataset Structure

Your dataset will contain:
- **Front camera**: RGB images (480x640x3)
- **Rear camera**: RGB images (480x640x3)
- **Actions**: linear.vel, angular.vel
- **State**: battery.level, orientation.deg, linear.vel, angular.vel

### Dataset Upload

Locally, your dataset is stored in: `~/.cache/huggingface/lerobot/{repo-id}`

At the end of recording, the dataset is automatically uploaded to your Hugging Face page:

```bash
echo https://huggingface.co/datasets/${HF_USER}/earthrover-navigation
```

Your dataset will be tagged with `LeRobot` for community discovery.

### Tips for Gathering Data

1. **Start Simple**: Begin with basic navigation tasks like moving from point A to point B
2. **Consistent Environment**: Keep lighting and background consistent during initial recordings
3. **Multiple Episodes**: Record at least 50+ episodes for robust training
4. **Smooth Controls**: Use smooth, consistent movements - avoid jerky motions
5. **Camera Coverage**: Ensure obstacles and targets are visible in camera views
6. **Add Variation Gradually**: Once you have good baseline performance, add more environment variations

For more guidance, check out our [blog post on dataset quality](https://huggingface.co/blog/lerobot-datasets#what-makes-a-good-dataset).

## Replay an Episode

To verify your recordings, replay episodes to ensure the robot follows the recorded actions:

```bash
python examples/earthrover_mini/replay.py
```

Or use the API:

```python
from lerobot.datasets.lerobot_dataset import LeRobotDataset
from lerobot.robots.earthrover_mini_plus import EarthRoverMiniPlus, EarthRoverMiniPlusConfig

# Load dataset
dataset = LeRobotDataset("your-username/earthrover-navigation")

# Initialize robot
robot = EarthRoverMiniPlus(EarthRoverMiniPlusConfig())
robot.connect()

# Replay episode 0
episode_index = 0
for idx in range(len(dataset)):
    if dataset.episode_data_index["episode_index"][idx] == episode_index:
        action = dataset[idx]["action"]
        robot.send_action(action)
        time.sleep(1.0 / dataset.fps)

robot.disconnect()
```

## Train a Policy

Once you have a good dataset, train a policy using LeRobot's training framework. See our [Getting started with real-world robots](./il_robots) guide for detailed training instructions.

Example training command:

```bash
python examples/training/train_policy.py \
    --dataset.repo_id=your-username/earthrover-navigation \
    --policy.type=diffusion \
    --output_dir=outputs/train/earthrover \
    --job_name=earthrover_navigation \
    --wandb.enable=true
```

## Evaluate Your Policy

After training, evaluate your policy on the robot:

```bash
python examples/earthrover_mini/evaluate.py
```

This will:
1. Load your trained policy
2. Connect to the robot
3. Run the policy to generate actions
4. Execute actions on the robot
5. Record success metrics

## Troubleshooting

### SDK Connection Issues

If you can't connect to the SDK:

1. **Check SDK is running**:
```bash
curl http://localhost:8000/data
```

2. **Verify port**: Ensure nothing else is using port 8000

3. **Check logs**: Look at the SDK terminal output for errors

### Camera Issues

If camera feeds are black or missing:

1. **Check SDK cameras**: The SDK should be receiving camera data from the robot
2. **Verify endpoints**: Test `/v2/front` and `/v2/rear` endpoints manually
3. **Network latency**: Reduce FPS if you're experiencing lag

### Keyboard Not Responding

On Linux, if keyboard controls don't work:
- Set the `$DISPLAY` environment variable
- See [pynput limitations](https://pynput.readthedocs.io/en/latest/limitations.html#linux)

On Mac:
- Grant Terminal keyboard access permissions
- System Preferences > Security & Privacy > Input Monitoring

## Technical Specifications

- **Control Interface**: HTTP REST API via Frodobots SDK
- **Cameras**: Front + Rear (640x480 @ 30fps max)
- **Action Space**: Linear velocity + Angular velocity
- **Observation Space**: Camera images + Robot state (battery, orientation, velocities)
- **Communication**: Cloud-based (no direct USB/serial connection)
- **Latency**: Network-dependent (suitable for FPS ≤ 10Hz)
